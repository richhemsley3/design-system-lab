Installation & Setup

Get started with Design System Lab in your project. Available as CSS, React, and Angular packages.

Packages

The design system is distributed as four npm packages. Install only what you need.

@sds/tokens
Design tokens as CSS custom properties, JSON, and TypeScript objects. Colors, spacing, typography.
@sds/css
Extracted component CSS with BEM class names. 45 components, individually importable.
@sds/react
React component library with TypeScript. 47 components with typed props and forwardRef.
@sds/angular
Angular component library. 47 standalone components with inputs and outputs.

Quick Start

React

  1. Install the packages 
    npm install @sds/react @sds/css @sds/tokens
  2. Import the styles in your app entry point (e.g. main.tsx or App.tsx) 
    import '@sds/tokens/css';
import '@sds/css';
  3. Use components with typed props 
    import { Button, Card, Tag } from '@sds/react';

function App() {
  return (
    <Card>
      <Card.Header title="Dashboard" />
      <Card.Body>
        <Tag status="success">Active</Tag>
        <Button variant="primary" size="md">View Details</Button>
      </Card.Body>
    </Card>
  );
}

Angular

  1. Install the packages 
    npm install @sds/angular @sds/css @sds/tokens
  2. Add styles to your angular.json 
    "styles": [
  "node_modules/@sds/tokens/css/index.css",
  "node_modules/@sds/css/dist/index.css",
  "src/styles.css"
]
  3. Import standalone components in your modules or components 
    import { SdsButtonComponent, SdsCardComponent } from '@sds/angular';

@Component({
  standalone: true,
  imports: [SdsButtonComponent, SdsCardComponent],
  template: `
    <sds-card>
      <sds-card-header title="Dashboard" />
      <sds-card-body>
        <sds-button variant="primary" size="md">View</sds-button>
      </sds-card-body>
    </sds-card>
  `
})
export class DashboardComponent {}

CSS Only

  1. Install tokens and CSS 
    npm install @sds/css @sds/tokens
  2. Import in your stylesheet 
    @import '@sds/tokens/css';
@import '@sds/css';
  3. Use BEM class names in your markup 
    <button class="sds-btn sds-btn--primary sds-btn--md">Submit</button>

<div class="sds-card">
  <div class="sds-card-header">
    <div class="sds-card-title">Dashboard</div>
  </div>
  <div class="sds-card-body">
    <span class="sds-tag sds-tag--success">Active</span>
  </div>
</div>
Selective imports: For smaller bundles, import only the components you need instead of the full bundle:

@import '@sds/css/components/buttons';
@import '@sds/css/components/cards';

Using Tokens

The token system uses CSS custom properties with the --sds- prefix. Import the token stylesheet to access all variables.

CSS Custom Properties

@import '@sds/tokens/css';

.my-component {
  color: var(--sds-text-primary);
  background: var(--sds-bg-surface);
  padding: var(--sds-space-400);
  font-size: var(--sds-type-body);
  font-weight: var(--sds-weight-semibold);
  border-radius: 8px;
  border: 1px solid var(--sds-border-default);
}

TypeScript Objects

For programmatic access (theming, charting libraries, CSS-in-JS), import the typed token objects.

import { colors, spacing, typography } from '@sds/tokens';

colors['--sds-color-blue-750']     // '#013D5B'
spacing['--sds-space-400']           // '16px'
typography['--sds-type-body']        // '14px'

JSON Exports

For build tools, design token pipelines, or Figma sync:

import colors from '@sds/tokens/json/colors';
import semanticColors from '@sds/tokens/json/semantic-colors';

Token Architecture

The design system uses a three-layer token architecture. Always use semantic tokens in your components — they automatically adapt to dark mode.

LayerExampleUsage
--sds-color-blue-750 Primitive Raw color value. Rarely used directly — reference via semantic tokens.
--sds-interactive-primary Semantic Maps to blue-750 in light mode, adjusts in dark mode. Use this in components.
--sds-bg-page Semantic Page background. White in light mode, dark gray in dark mode.
--sds-text-primary Semantic Primary text color. Warm gray in light mode, cool gray in dark mode.
--sds-space-400 Space 16px. Use for standard component padding.
--sds-type-body Typography 14px. Primary body text size.
Dark mode: The token system supports dark mode via prefers-color-scheme: dark. Semantic tokens automatically swap warm grays for cool grays — no code changes needed in your components.

Font Setup

The design system uses Proxima Nova (weights 300-800). This is a licensed font — your team must provide their own license.

/* Load Proxima Nova via @sds/tokens typography */
@import '@sds/tokens/css/typography';

/* Or self-host and reference the font-family token */
body {
  font-family: var(--sds-font-sans);
  /* 'Proxima Nova', -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif */
}

If Proxima Nova is unavailable, the system falls back to the platform's default system font.

Component Conventions

All components in both React and Angular follow the same patterns.

CSS Class Naming

Components use BEM naming with the .sds- prefix:

/* Block */
.sds-alert { }

/* Element */
.sds-alert__icon { }
.sds-alert__content { }

/* Modifier */
.sds-alert--warning { }
.sds-alert--compact { }

React Components

Every React component supports className for composition and ref forwarding.

import { Button } from '@sds/react';

// All props are typed
<Button
  variant="primary"
  size="md"
  className="my-custom-class"
  onClick={handleClick}
  ref={buttonRef}
>
  Submit
</Button>

Angular Components

Angular components use standalone components with typed inputs and outputs.

import { SdsAlertComponent } from '@sds/angular';

// In your component
<sds-alert variant="warning" title="Attention" dismissible (dismissed)="onDismiss()">
  Check your configuration.
</sds-alert>

Project Structure

@sds/tokens          Design tokens — CSS, JSON, TypeScript
  css/               colors.css, spacing.css, typography.css
  json/              colors.json, semantic-colors.json
  dist/              TypeScript objects (ESM + CJS)

@sds/css             Component stylesheets
  dist/
    base.css         Reset + body defaults
    index.css        Full bundle (all components)
    components/      Individual component CSS files

@sds/react           React component library
  dist/
    index.mjs        ESM bundle
    index.js         CJS bundle
    index.d.ts       TypeScript declarations

@sds/angular         Angular component library
  dist/
    fesm2022/        ES module bundle
    esm2022/         Individual component modules
