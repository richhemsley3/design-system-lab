Selection Controls

Advanced selection patterns for checkboxes, radios, and toggles. These compound components build on the base form inputs to handle multi-select lists, card selectors, settings toggles, filter chips, and table row selection.

Checkbox Group

A vertical stack of checkboxes with an optional parent "Select all" that shows an indeterminate state when some children are checked. Used for permission sets, data classification selection, and bulk operations.

Radio Cards

Bordered cards with radio selection for choosing between mutually exclusive options. Each card has a title, description, and optional icon. The radio indicator leads for scanability, followed by the icon and content. Use for policy type selection, scan mode configuration, or plan selection.

Checkbox Cards

Multi-select card pattern for choosing one or more options. Same layout as radio cards but supports multiple selections. The checkbox indicator leads the row for consistent scanability. Use for data source selection, notification channel configuration, or feature enablement.

Interactive

Toggle List

A vertical list of settings-style toggle rows with labels and descriptions. The entire row is clickable and highlights on hover to communicate interactivity. Supports optional section grouping for organized settings pages. Use for feature flags, notification preferences, and security settings.

Filter Chips

Pill-shaped toggleable buttons for filtering views. Available in single-select (radio) and multi-select (checkbox) variants. Touch targets meet the 44px WCAG minimum via invisible hit area expansion. Use for dashboard filters, log severity levels, and status filtering.

Inline Checkbox Table

Table rows with leading checkboxes for bulk operations. A selection action bar appears when one or more rows are selected, showing the count, a "Deselect all" link, and contextual bulk actions with bordered buttons for clarity. Use for data tables requiring multi-row operations.
