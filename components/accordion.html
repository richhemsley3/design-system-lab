Accordion

Expandable sections for organizing related content into collapsible panels. Used in settings views, preference panels, and documentation sections.

Basic Accordion

Only one section can be open at a time. Clicking a header closes any previously open section. Uses data-accordion-exclusive to enforce single-expand behavior.

Exclusive mode — single open
Define how long records are retained before automatic archival. Retention periods can be configured per category, with separate rules for production, staging, and archival environments.
Configure security settings and credential rotation schedules. Supports multiple authentication methods with automatic expiration and renewal policies.
Manage role-based access control (RBAC) settings that govern who can view, modify, or delete resources. Assign granular permissions at the team, project, or individual level.
Track all user actions and system events with detailed activity logs. Logs are immutable once written and can be exported for reporting and analysis.

Multi-expand Accordion

Multiple sections can be open simultaneously. Omit data-accordion-exclusive to allow independent toggling of each section.

Independent expand — multiple open
Configure network rules, IP allowlists, and connection settings. Network policies ensure that production workloads are isolated from development traffic.
Connect SAML 2.0 and OIDC identity providers for single sign-on. Supports Okta, Azure AD, Google Workspace, and custom IdP configurations with just-in-time user provisioning.
Set session timeout durations, concurrent session limits, and idle lockout rules. Administrators can force-terminate active sessions for specific users or across the entire organization.

Accordion with Icons

Section headers include a leading Material Symbol icon for additional visual context. Use .sds-accordion-leading-icon before the header text.

Leading icons
Real-time monitoring of data access patterns to detect anomalous behavior. Machine learning models flag suspicious queries, bulk exports, and privilege escalation attempts.
Map organizational controls to your compliance requirements. Automated reporting generates review-ready reports on demand.
Automatically categorize and label items across all connected sources. Categories include personal, financial, internal, and confidential with configurable priority tiers.
Configure notification rules for security events. Route alerts to Slack, PagerDuty, email, or webhooks based on severity level and affected resource type.

Nested Accordion

Accordion items can contain child accordions for deep hierarchies. Useful for multi-level settings like region-specific compliance configurations.

Two-level hierarchy
Configure where data is stored based on geographic and regulatory requirements.
All personal data for EU subjects must be stored in Frankfurt or Dublin regions. Cross-border transfers require Standard Contractual Clauses.
California consumer data is stored in US-West regions. Opt-out requests must be processed within 15 business days per CCPA regulations.
Singapore and Australian data residency requirements mandate local storage with cross-border transfer agreements for inter-regional processing.
Configure automated backup schedules, retention windows, and disaster recovery failover settings across all connected services.

Bordered Variant

Each item has visible borders and dividers for a more structured appearance. Apply .sds-accordion--bordered to the container.

Bordered with dividers
Enforce minimum length, complexity rules, and password history depth. Passwords must contain at least 12 characters with uppercase, lowercase, numeric, and special character requirements.
Require a second factor for all user logins. Supported methods include TOTP authenticator apps, hardware security keys (FIDO2/WebAuthn), and SMS verification codes.
Generate, rotate, and revoke API keys for programmatic access. Keys can be scoped to specific endpoints and assigned rate limits per consumer application.
Restrict access to known IP addresses or CIDR ranges. Requests from unlisted addresses are blocked and logged for security review.
